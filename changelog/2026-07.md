# Update Log July 2026

Since the [December 2025 update](2025-12.md) introduced the Artus Talos and Artus Scorpion end effectors, development has focused on stabilizing `ArtusAPI_V2`, retiring the legacy v1 API entirely, and extending hardware support to the Artus Dex.

## List of Updates

### API
* Legacy `ArtusAPI` (v1) has been fully removed from the repository — `ArtusAPI_V2` is now the only supported entry point, and all examples reference it exclusively.
* Docstrings standardized to Google style (Args/Returns/Raises) across the ArtusAPI package and examples.
* `set_joint_angles_by_name` no longer relies on dictionary indices — it resolves joints by name only.
* `get_*` feedback functions now return dictionaries instead of raw lists.
* Added Ctrl+C (SIGINT) handling so the hand is sent to sleep gracefully on exit instead of losing power mid-motion.
* Fixed `wait_for_ready` to return correctly given all acceptable actuator states.
* Added trajectory acknowledgement to feedback data while in active control mode.

### Communication
* Migrated RTU communication from `minimalmodbus` to `pymodbus`.
* Added Modbus TCP support alongside RTU.
* Added support for individual, per-hand Modbus slave IDs (previously fixed per robot/hand type).
* Improved error code management — drivers now clear and report errors more reliably.

### Hardware Support
* Added initial support for the **Artus Dex**.
* Artus Lite migrated onto the new Modbus communication structure with cascading PID control.
* Artus Scorpion: improved force output, added debug feedback, and corrected the reported number of contactile sensors.
* Added a config command to edit WiFi name/password on Artus Lite (firmware v10.0.0).
* Added voltage and temperature feedback.

### GUI
* Added speed/force control inputs.
* Added save/load grasp functionality.
* Graph axes now scale min/max dynamically.
* Fixed UI rendering issues.
* Fully tested with the Artus Talos hand (per-joint feedback colors, position commands).

### Examples
* Added a generalized **Standard Bots** demo ([`examples/StandardBotsDemo/`](../examples/StandardBotsDemo/)).
* MediaPipe teleoperation: switched to multiprocessing for improved responsiveness, integrated calibration into startup, and removed a stray print statement that was breaking the hand-control process.
* Removed the Kalman filter from the Google MediaPipe teleop pipeline.
* Integrated Contactile force sensor output testing into the Lite API examples.
* `general_example.py` now accepts a control-type input for the v2 API.
* Added a file-location option for flashing via the CLI.

### Configuration & Requirements
* Fixed `requirements_v2.txt` for pip installs.
* Config file now has retry logic on invalid input (quality-of-life).

## Additional Notes
* As with the December release, this is not yet pushed to PyPI — pull the latest repo to get these changes.
